# Інструкція з Запуску Застосунку

## Швидкий Старт

### 1. Встановлення залежностей

```powershell
# Створити віртуальне середовище (якщо ще не створено)
python -m venv .venv

# Активувати віртуальне середовище
.venv\Scripts\Activate.ps1

# Встановити залежності
pip install -r requirements.txt
```

### 2. Запуск застосунку

```powershell
# Переконайтеся, що віртуальне середовище активоване
.venv\Scripts\python.exe -m schedule_askue.main
```

Застосунок запуститься з GUI вікном. Логи виводяться:
- У консоль (stdout)
- У файл `logs/app.log`

## Перевірка Роботи

```powershell
.venv\Scripts\python.exe -m unittest discover -s tests -v
```


## Логування

### Рівень логування

За замовчуванням: **INFO**

Логуються:
- Старт/завершення генерації графіка
- Кількість працівників, побажань, правил
- Результати валідації (помилки/попередження)
- Критичні помилки UI

### Перегляд логів у реальному часі

```powershell
Get-Content logs\app.log -Wait -Tail 50
```

## Troubleshooting

### Проблема: Застосунок не запускається

**Симптом:** Помилка при запуску, вікно не відкривається

**Рішення:**
1. Перевірте `logs/app.log` для деталей
2. Переконайтеся, що всі залежності встановлені:
   ```powershell
   pip install -r requirements.txt
   ```
3. Перевірте, що база даних існує:
   ```powershell
   Test-Path schedule.db
   ```

### Проблема: Worker timeout (генерація зависає)

**Симптом:** Генерація не завершується, через 40 секунд з'являється помилка

**Причини:**
- Занадто складна комбінація правил і побажань
- Конфлікти між обов'язковими побажаннями
- Багато працівників (>10)

**Рішення:**
1. Спростіть правила
2. Зменшіть кількість обов'язкових побажань
3. Перевірте логи для деталей конфлікту

### Проблема: AttributeError при запуску

**Симптом:** `AttributeError: 'MainWindow' object has no attribute 'balance_tab'`

**Рішення:** Ця проблема виправлена в поточній версії. Переконайтеся, що ви використовуєте останню версію коду.

### Проблема: Логи не з'являються в консолі

**Симптом:** У консолі немає виводу, тільки у файлі

**Рішення:** Ця проблема виправлена. Тепер логи виводяться і в консоль, і у файл.

## Структура Проєкту

```
schedule_askue/
├── core/           # Бізнес-логіка
│   ├── generator.py           # Фасад генерації
│   ├── priority_scheduler.py  # Алгоритм генерації (~2000 рядків)
│   ├── validator.py           # Валідація графіка
│   └── ...
├── db/             # База даних
│   ├── models.py              # Доменні моделі
│   └── repository.py          # SQLite CRUD
├── ui/             # Інтерфейс PyQt6
│   ├── main_window.py         # Головне вікно
│   ├── schedule_tab.py        # Вкладка графіка
│   └── ...
├── export/         # Експорт
│   ├── excel_exporter.py
│   └── pdf_exporter.py
└── main.py         # Точка входу

config.yaml         # Конфігурація
schedule.db         # База даних SQLite
logs/app.log        # Логи
```
